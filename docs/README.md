## üìñ Documentation Guidelines

When adding new documentation:

1. **Place files in the appropriate subdirectory** based on audience:
   - `guides/` - For end users
   - `development/` - For developers
   - `architecture/` - For system design
   - `integration/` - For external integrations
   - `deployment/` - For deployment guides
   - `security/` - For security documentation
   - `reviews/` - For analysis documents

2. **Update this README** with links to new documentation

3. **Follow the existing format** - Use clear headings, code examples, and cross-references

4. **Keep documentation up to date** - Update docs when features change
